영역을 나누는 형식(성질)

    
block 형식

    - 공간을 수직 분할
    -> 수직으로 되어있는 화면을 가로로 잘라서
        여러 행을 만드는 것
    
    - 크기를 지정하는 css 속성인 width, height를 사용할 수 있다!

    ex) h1~h6 (제목 영역을 분할)
        p/pre (문단 분할)
        hr (수평선을 그어 화면 분할)
        div (영역 분할)
        시맨틱 태그(div 태그에 이름을 넣은 태그)
            -> header, main, footer, section, article, aside
  
      
inline 형식

      - 공간을 수평 분할
      -> 수평으로 되어있는 화면을 세로로 잘라서
          여러 열(컬럼)을 만드는 것

      - 글자랑 같은 속성을 갖고있음
      - 크기를 지정하는 css 속성인 width, height를 사용할 수 없다...(영역이기에!)
      
      ex) b/strong, i/em, mark, span, img

block(div 태그) / inline(span 태그) 차이점 1 : 줄바꿈

block(div 태그) 확인

    block 형식은 작성된 내용이 짧더라도 
    한 줄을 모두 차지하려는 성질 때문에
    다음 내용은 다음 줄에 작성이 된다.
첫 번째 영역
두 번째 영역
세 번째 영역

inline(span 태그) 확인

    inline 형식은 작성된 내용만큼의 크기만을
    차지하는 성질을 갖고있어
    다음 내용이 바로 옆(오른쪽)에 이어서 작성된다
첫 번째 영역 두 번째 영역 세 번째 영역

block(div 태그) / inline(span 태그) 차이점 2 : 영역 지정 방식

  • block : 사각형 박스 형태로 영역을 지정
  • inline : 내부에 작성된 내용(content) 단위로 영역을 지정

block(div 태그) / inline(span 태그) 차이점 3 : 크기 지정

  • block : 크기 지정을 width/height로 할 수 있음
  • inline : 크기 지정을 width/height로 할 수 없음

iframe 태그

    - 웹 문서 내부에
    또 다른 웹 문서를 추가하는 태그(inline 형식)

    - 웹 문서를 겹쳐서 보여주기 때문에
        활용 방법에 따라 다양한 연출이 가능한 장점 있음

    - 하지만, 한 번에 여러 페이지를 로딩하기 때문에
        컴퓨터 성능, 인터넷 속도에 따라 느려지거나, 오류가 발생할 수 있다.
